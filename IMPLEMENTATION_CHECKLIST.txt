DATABASE SCHEMA:
  Table: Categories
  Columns:
    - Id (INT, PK, IDENTITY)
    - Name (NVARCHAR(50), NOT NULL, UNIQUE)
    - Description (NVARCHAR(200), NULL)
    - IsActive (BIT, NOT NULL, DEFAULT 1)
    - CreatedAt (DATETIME2, NOT NULL)
    - UpdatedAt (DATETIME2, NULL)
    - DisplayOrder (INT, NOT NULL, DEFAULT 0)
    - IconUrl (NVARCHAR(MAX), NULL)
    - PriceMultiplier (DECIMAL(18,2), NOT NULL, DEFAULT 1.0)

DESIGN PATTERNS:
  • Repository Pattern
  • Dependency Injection
  • DTO Pattern
  • RESTful API Design

VALIDATION RULES:
  • Name: 2-50 chars, unique, required
  • Description: 0-200 chars, optional
  • Display Order: 0-1000, required
  • Price Multiplier: 0.1-10.0, required
  • Icon URL: Valid URL, optional
  • Is Active: Boolean, required
